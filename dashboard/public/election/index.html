方法论说明

本 dashboard 的核心任务是将地缘政治冲击（霍尔木兹海峡封锁）通过宏观与政治传导链转换为美国选举结果预测。下文逐项说明所用模型、选择理由、历史预测准确度、传导链结构与数据来源。

一、为什么选这四个模型？为什么没选其他模型？

美国选举预测领域约有 6-8 个有学术背书的模型。本 dashboard 挑选四个互补模型（Fair / Hibbs / Abramowitz / Lewis-Beck-Tien）并构建集合预测。Lewis-Beck 是 2026-05 PR 加入的第 4 个模型——前 3 个都是 pocketbook / retrospective 谱系（GDP 增长、实际收入、支持率），全部内生于宏观经济变量；Lewis-Beck 加入 sociotropic（"国家失业率 + 通胀"）维度，对 1982 / 2010 这种失业率主导的情景提供独立信号。理由如下：

入选四个模型的具体公式

1. Fair Vote Equation（Ray Fair, Yale 1978），预测现任党两党票份额：

V_民主党 = 47.42 + 0.667·G·I + (-0.690)·P·I + 0.984·Z·I
        + 3.41·DPER + (-3.46)·DUR·I + (-0.105)·I

参数定义：
  G    = 选举年 Q1-Q3 实际人均国内生产总值（GDP）增长率（年化 %）
  P    = 任期内国内生产总值平减指数 15 季度增长率绝对值（年化 %）
  Z    = 任期内 15 季度中实际人均增长 >3.2% 的"好消息季度"数量（最大 15）
  DPER = 现任总统亲自参选系数（D=+1, R=-1, 开放席位=0）
  DUR  = 在位时长系数（首任=0, 二连任=1, 三连任=1.25...）
  I    = 党派虚拟变量（D=+1, R=-1）
均方根误差（RMSE）= ±2.5pp（全国两党票出样本）

2. Hibbs 面包与和平模型（Douglas Hibbs, 1982/2008），预测现任党两党票份额：

V_现任党 = 46.10 + 3.49·R + (-0.00355)·累计战争伤亡

参数定义：
  R = λ-加权 15 季度实际人均可支配收入（RDPI）增长率（年化 %）
  λ = 0.915（按季度几何衰减——最近季度权重最大）
  累计战争伤亡 = 任期内非主动战争中美军死亡人数（2026-2028 期间为 0）
均方根误差（RMSE）= ±1.85pp（全国两党票，1952-2012 历史样本）

3. Abramowitz Time-for-Change（Alan Abramowitz, 1988），预测现任党两党票份额：

V_现任党 = 51.40 + 0.108·G_Q2 + 0.642·NETAPP + (-4.0)·TERM2

参数定义：
  G_Q2    = 选举年第二季度实际国内生产总值（GDP）增长率（年化 %）
  NETAPP  = 现任总统 6 月净支持率（赞同率 - 反对率）
  TERM2   = 现任党连续执政是否 ≥2 任的虚拟变量（是=1, 否=0）
均方根误差（RMSE）= ±1.90pp

4. Lewis-Beck-Tien Sociotropic Misery Index（2026-05 加入），预测现任党两党票份额：

V_现任党 = 53.0 - 0.85 · (UNRATE + CPI_YoY)

参数定义：
  UNRATE  = U-3 失业率（%）
  CPI_YoY = 头条 CPI 同比通胀率（%）
  misery  = UNRATE + CPI_YoY（"痛苦指数" sum, 经典 Lewis-Beck-Tien 1996/2008 形式）
均方根误差（RMSE）= ±2.50pp（[CALIBRATION_PENDING] — 当前系数为 1980/2010 两点 prior，未在 1948-2020 全样本上 OLS 拟合）

为什么加入：Fair/Hibbs/Abramowitz 都属 pocketbook 谱系（个人收入），无法捕捉 1982 (UNRATE 10.8% / CPI 3.8%) 与 2010 (UNRATE 9.6% / CPI 1.1%) 这种"失业率主导"的中期选举。Lewis-Beck 是 sociotropic 谱系（看国家整体经济新闻），添加独立的失业率维度。详见 Mutz (1992) 与 Lewis-Beck & Tien (1996, 2008) 关于 pocketbook vs sociotropic 投票的文献。

5. Bafumi-Erikson-Wlezien（BEW, 2010），预测中期众议院席位变化：

现任党席位变化 = -22 + 3.5·现任党通用国会选票领先 + 0.4·现任总统净支持率

参数定义：
  现任党通用国会选票领先 = 通用国会选票（Generic Ballot）现任党 - 在野党，pp 单位
  现任总统净支持率 = 赞同率 - 反对率，pp 单位
  -22 截距 = 历史中期选举现任党平均失席位
均方根误差（RMSE）= ±12 席

四个模型使用不同输入变量：Fair 综合考虑增长、通胀与结构因素；Hibbs 仅观察实际人均可支配收入但加权 15 季度记忆（注：2026-05 PR 后修正——之前误用总额 DSPIC96，现切到人均 A229RX0，符合 Hibbs 1982/2008 原始 specification）；Abramowitz 主要看支持率（民意层）；Lewis-Beck 加入失业率 + 通胀 sociotropic 维度。Fair / Hibbs / Abramowitz 是 pocketbook 谱系（个人收入），Lewis-Beck 是 sociotropic 谱系（国家整体经济新闻）——两个谱系的对冲让 ensemble 不再单边偏向 retrospective 视角。四模型预测一致时可信度高；分歧大时（如当下 Fair 与 Abramowitz 差 11.7pp）则提示经济数据与选民感知出现严重背离，1992 / 2016 / 2024 选举均属此种情形。

为什么没选其他常见模型？

未选模型排除理由
Lichtman 13 Keys to the White House 方法论：13 个二元判断（每个由分析师主观打分）。学界对其科学性争议大——2024 年预测哈里斯赢哈里斯（Harris）是其首次失败，揭穿"必胜"神话。排除原因：变量本质是定性主观判断，不可机械计算；不能纳入自动化 dashboard。
Norpoth Primary Model（Helmut Norpoth） 方法论：仅用初选表现 + 钟摆效应。声称 1912 年以来 25 次预测对 22 次。2016/2024 都正确预测特朗普赢。排除原因：变量选择被怀疑过拟合——25 次中错过几次的模式不可重复验证；初选数据 2027 年才有，对 2028 当下无前瞻价值。
Drew Linzer Dynamic Bayesian Model（2013） 方法论：贝叶斯州层模型 + Kalman filter 实时更新。Economist 2020 模型、Silver Bulletin 都用类似架构。排除原因：需要每周民调数据 + 州层模型，数据需求量大；本 dashboard 走的是"基本面 + 情景分析"路径，不是民调驱动。Linzer 类模型适合选举前 6 个月的预测窗口，不适合距大选 2.5 年的当下 2028。
Justin Grimmer 等人的"朴素预测"（2024） 方法论：用"上届结果 + 现任党"作基线，论证学术模型在统计上不优于此基线。排除原因：本身是元批评，不是预测工具；其结论已通过我们的"模型分歧 = 警报"机制吸收。
Allan Lichtman 之外的其他经济模型（如 Cuzán、Holbrook） 方法论：与 Fair / Hibbs / Abramowitz 高度相似的回归模型。排除原因：与已选三个模型的输入变量重叠度高，无法增加集合预测的方差降低收益（PollyVote 论文证明：相关性高的模型加入集合反而效果有限）。

选择原则总结：(1) 学术背书 + 公开公式可复现；(2) 输入变量分散（综合、收入、支持率），互补性最大化集合预测增益；(3) 当前数据可获取（不依赖未发布的初选数据）；(4) 出样本均方根误差（RMSE）≤2.5pp。

二、集合预测的方法、已知局限、与 B3 backtest 的修正

原理：单一模型必有偏好；集合互补模型可降低平均误差。德国学者 Andreas Graefe 提出的 PollyVote 方法证明：将所有公开模型按"反向 RMSE"加权平均（历史误差小的模型权重大），出样本 RMSE 比任何单一模型低 30-40%。

本 dashboard 采用的具体计算（2026-05 PR 升级到 4-way ensemble）：

权重 w_i = (1 / RMSE_i) / Σ(1 / RMSE_j)
集合预测 = Σ w_i × 模型_i 预测值
集合方差 = Σ w_i² × RMSE_i² （假设独立性 — 见下方 ⚠️）

当下权重（4-way ensemble，按 paper-reported RMSE 反比）：

  • Hibbs ≈ 29%（paper RMSE 1.85）
  • Abramowitz ≈ 28%（paper RMSE 1.90）
  • Fair ≈ 21%（paper RMSE 2.50）
  • Lewis-Beck ≈ 21%（paper RMSE 2.50 [CALIBRATION_PENDING]）

独立性假设下集合预测带宽约 ±1.07pp（按生产代码计算）。

⚠️ 这种计算方法的已知问题：上述方差公式假设四个模型相互独立，但实际上 Fair / Hibbs / Abramowitz 都使用了重叠的宏观经济变量（GDP 增长、通胀、收入），Lewis-Beck 与 Hibbs 在失业率周期年共同 hit outlier。模型预测误差是正相关的，而非独立。

为什么 backtest 不自动改 config.py 系数

  • n=11 太小——4 模型各 1-6 个系数，自由度 5-10，估计噪音大。Fair β_dper 在 OLS 输出中符号翻转（3.41 → -6.82），这是小样本 + collinearity artefact，不是真实信号。
  • 2020 COVID 是 extreme outlier——Q2 GDP -31.4% 严重扭曲 Abramowitz 系数估计。完整 1948-2020 18 选举样本会更平衡。
  • 当前实施：backtest 写到 data/election/backtest_results.json（手动 trigger only），methodology panel 显示比较数据让读者看到 paper RMSE 与样本 RMSE 的 gap，但生产 ensemble 仍用 paper 系数。
  • 扩展路径：填补 1948-1976 的 7 次选举到 HISTORICAL_ELECTIONS list（需 Hibbs 1982 spreadsheet + Fair archive + Gallup historical），单 PR 即可。

模型分歧本身是信号

当四个模型预测一致（说明输入信号强烈），集合预测可信度高。当四模型分歧大（如当下 Fair 与 Abramowitz 差 11pp），表明模型在"权重不同输入变量时给出不同结论"——这本身就是诊断信号，意味着经济数据与选民感知出现严重背离，1992 / 2016 / 2024 选举均属此种情形。2026 dashboard 显示的 Fair vs Abramowitz gap 是一个 attention signal，不是单纯的 noise。

为什么 ICS↔approval 标 parallel 而不是单向链

原 9 步链把 step 7 (ICS) → step 8 (approval) 当单向因果（-10 ICS → -1.1pp approval）。这是 misleading 的：

  • Erikson, MacKuen & Stimson (2002) The Macro Polity 第 4 章用 VAR 分析说明：approval → ICS 的 Granger 因果性强于反向（政治情绪溢出到对经济的感知）；两者主要由基本面（real income + 失业率 + 物价水平）共同驱动。
  • De Boef & Kellstedt (2004) 同样论证：把 ICS → approval 建成单向链会双重计算同一冲击。
  • 2026-05 PR 修复：transmission_state.json 加 parallel_group 字段，前端 chain tab 把 step 7+8 用紫色边框 + "‖" badge 标 parallel，并加 indigo notice 解释为什么读它们应该读作 co-driven signals 而不是 chain steps。
  • 对 Hibbs 1992 / 2016 / 2024 失灵的解释：这三次大选 ICS 与 RDPI 严重背离——RDPI 数据 ok 但 ICS 跌（1992 Bush 41 / 2024 Biden）或 RDPI 弱但 ICS 上升（2016 Clinton）。当 perception 与 income 背离时，Hibbs（用 RDPI）系统性 miss——这是 perception-basement 双信号同时观察的必要性。

四、军事干预的额外传导通道（情景 D 专用）

第三节描述的九步传导链覆盖经济通道——油价通过通胀和实际收入影响选民感知。但情景 D（美军直接控制海峡）涉及军事行动，触发两个完全独立的非经济通道。本节解释这两个通道的工作机制，以及为什么"军事胜利"对参议院、众议院、总统大选的影响截然不同。

通道 A：Rally-around-the-flag（爱国情绪聚集）

机制：当美国进行重大军事行动时，公众情绪短期内出现爱国主义聚集，对总统支持率产生瞬时正向冲击，即便许多选民平时不支持总统。

历史校准（来自 Mueller 1973 经典论文 War, Presidents and Public Opinion）：

  • 1979 年 11 月 Iran 人质危机：Carter 支持率 +18pp
  • 1991 年 1-2 月 Desert Storm：Bush 41 支持率从 60% 涨到 89%（+29pp 峰值）
  • 2001 年 9 月 9/11 事件：Bush 43 支持率 +35pp（历史峰值）

本 dashboard 对"成功护航 / 控制海峡"取 +8pp（小型军事行动的中位估计，远低于全面战争）。

关键特征：rally 是脉冲信号，会衰减。Mueller 测得半衰期约 6 个月——意思是每过 6 个月，剩余强度减半。下表展示衰减过程：

距事件月数剩余强度对应支持率提升
0（事件刚发生）100%+8pp
6（一个半衰期）50%+4pp
12（两个半衰期）25%+2pp
30（2026 大选）3%+0.2pp（基本归零）

历史镜像：Bush 41 在 Desert Storm 后支持率 89%，但 18 个月后跌至 34%——克林顿因此赢得 1992 大选。这是 Fair 模型最著名的失败案例：经济数据看起来不错，模型预测 Bush 赢 55%，但 rally 早已衰减，选民只记得衰退。

通道 B：Casualty drag（伤亡累积）

机制：长期军事部署不可避免产生美军阵亡，每位阵亡士兵都在选民记忆中累积负面权重。Hibbs Bread & Peace 模型把这个变量正式纳入：每位阵亡 ≈ -0.00355pp 选票（系数来自 1952-2012 历史数据回归）。

历史校准：

  • Bush 41 海湾战争：~150 美军阵亡（短期，效应有限）
  • Bush 43 阿富汗战争：累积 ~2400 美军阵亡（拖累共和党 2008 / 2012 选举）
  • 霍尔木兹护航行动：估计远低于战争，本 dashboard 对情景 D 取 -3pp 长期拖累

关键特征：伤亡是累积信号，不衰减。每位阵亡都进入永久记账，永远不忘。下表展示累积过程：

距事件月数累积比例对应支持率拖累
00%0pp
6（2026 中期）50%-1.5pp
30（2028 大选）100%-3pp（全部累积）

两个通道相加 = 时间剪刀差

把 rally（衰减）和伤亡（累积）叠在一起，会得到一个反直觉的曲线：军事干预对短期选举有帮助，但对长期选举反而是负的。具体到 dashboard 的两个时点：

时点Rally 残留伤亡累积净支持率影响

2028 大选（30 个月后）+0.2pp（基本归零）-3.0pp（全部累积）-2.8pp（共和党受损）

对照基线情景（B 拖延）：D 控制场景下油价是 $90/桶（基线 B 是 $100），所以经济通道对共和党有利（油价低 → 通胀低 → 实际收入好）。但 rally 衰减+伤亡累积的非经济通道把这个经济好处抵消了。

为什么对参议院、众议院、总统的影响不同

同样的支持率变化，三个层级响应不一样。原因是输入变量、响应公式和结构限制都不同：

层级输入指标结构限制D 控制 vs B 基线效应
2026 参议院 Generic Ballot（推算） Class II 结构 22R 防守 vs 11D，硬上限 D 净增 ≤4 席 D 多数概率 60% → 53%（rally 帮 R 守住 ME / NC 等关键席位）
2026 众议院 Generic Ballot + 净支持率（双输入） 全 435 席改选，无结构限制 R 失 39 席 → 失 36 席（rally 帮 R 守 3 席）
2028 总统 Q2 GDP + 6 月支持率 + 任期变量 2028 是开放席位，无现任优势 R 票 48.6% → 48.4%（rally 已衰减，伤亡抵消油价好处）

反直觉发现：D 控制场景在参议院的百分比效应（D 多数概率 -7pp）比众议院更明显。这是因为参议院 33 席规模小、ME / NC / TX 等摇摆席位都在边缘——支持率小幅上涨就能让 R 多守 1-2 席。众议院 435 席规模大，几个 pp 的支持率变化被结构稀释。

  • Rally 和伤亡这两个通道都对所有 35 个参众议席候选人同等影响——模型假设候选人外交立场是中性的。但真实世界候选人立场各异（鹰派 vs 鸽派），可能产生差异化效应。
  • Rally 系数 +8pp 是中位估计，sensitivity sweep 提供 +5 到 +15 区间但不包括 +24pp 这种 1991 GW 峰值——如果美军快速取胜伤亡极少，rally 可达 +24pp。
  • 伤亡系数 -3pp 是基于"成功护航"假设。如果实际是 Iraq War 级别的部署，high_casualty calibration 用 -8pp，但还可能更深（Iraq 累计 -10pp 以上）。
  • 本 dashboard 不建模候选人质量、初选动员、广告战、辩论表现等竞选过程因素——这些可能放大或抵消上述传导效应。
  • casualty_sensitivity 仅在预设 rally_effect_pp 的 Hormuz 情景（当前只有 D_us_control）上计算；E_total_war 应该也激活但尚未配置。

五、时间维度：为什么伊朗战争对 2026 中期影响小、对 2028 大选影响大

这是 dashboard 输出的一个重要发现，但容易被忽视：5 种霍尔木兹情景对 2026 中期选举的影响很小，但对 2028 大选影响巨大。下表展示这种时间维度上的不对称：

选举5 情景跨度含义
2026 众议院 共和党失席位-36 到 -40（4 席跨度）几乎无差别
2026 参议院 民主党净增席位+3.7 到 +4.0（0.3 席跨度）几乎无差别
2026 参议院 民主党多数概率53% 到 60%（7pp 跨度）较小差异
2028 总统 共和党两党票44.7% 到 49.7%（5pp 跨度）跨度大得多

三个原因

原因一：2026 中期的基本面已经定型。无论伊朗战争往哪个方向发展，下面这些指标已经反映在 dashboard 上：

  • Trump 净支持率 -13.5（已进入告警区）
  • 密歇根消费者信心 53.3（接近 1980 / 2008 / 2022 三次危机底部）
  • Generic Ballot 民主党 +3.5（已显示共和党中期不利）

这些指标已经把"Trump 第一年表现 + 当前 Hormuz 冲击的部分传导"打包定价。即便伊朗冲突明天彻底解决（情景 A），共和党在 2026 仍然会失约 40 席。政治伤害已经形成。

原因二：传导链时滞 4-9 个月，大部分赶不上 11 月中期。从 5 月 8 日到 2026 年 11 月 3 日中期日只有约 6 个月，而油价冲击的完整传导需要：

  • 油价（第 1 步）→ 当前已到位
  • CPI（第 4 步）→ 7-8 月才完整反映
  • 实际收入（第 6 步）→ 9 月才有数据
  • 消费者信心（第 7 步）→ 已经先行崩盘，但这是历史冲击的结果，不是新增冲击

中期之前能"吸收"的伤害已基本到位。任何在 5-11 月之间发生的新增油价冲击，对 11 月投票的边际影响有限。

原因三：参议院席位地图本身的硬约束。Class II 改选 22R / 11D 防守，民主党最多净增 2-4 席。这是结构性物理约束，不受任何油价情景影响。

真正的"伊朗战争效应"在 2028

把 2028 共和党两党票数字看清楚：A 缓解 49.7% → E 全面战争 44.7% = 5pp 跨度。这相当于：

  • 49.7%：共和党几乎追平民主党（虚拟选举团差距 <50 票）
  • 44.7%：共和党类似 1980 卡特 / 2008 麦凯恩级别的惨败（电学院差距 200+ 票）

两个完全不同的政治世界。这才是伊朗战争的真正赌注。

这对分析意味着什么

  • 不要再追问"伊朗战争对 2026 中期影响多大"——答案是"很小，因为基本面已定"。
  • 应该追问：当前 -13.5 支持率 + 53 ICS 是怎么形成的？6/9/10 月分别会更新哪些指标？2028 大选的 5pp 跨度赌的是什么？
  • 简言之：2026 中期已经是"民主党大概率拿下众议院 + 参议院僵持"的基线。伊朗战争已经变成 2028 的故事，不再是 2026 的故事。

需要警惕的盲点

模型抓不到非经济动员效应。1980 年伊朗人质危机激起爱国情绪短期帮 Carter（rally +18pp）。如果 2026 发生大规模美军伤亡 / 美国本土被袭击 / 类似 9/11 级别事件，整个图景会被重写——这些不在传导链里。如果未来 6 个月发生质变（不是延续而是新事件），所有数字都得重算。

六、关税通道（独立于油价 shock，2026-05 PR 新增）

原 dashboard 的传导链把通胀 shock 全部当成 外生（来自霍尔木兹海峡油价）。但 2025-2026 真实主导的通胀冲击源不是油价，是 Trump 第二任期关税——这是 dashboard 之前完全 miss 的通道。2026-05 PR 把关税作为独立通道加进 macro projection。

为什么关税要独立建模（不能合并进油价通道）

  • 外生 vs 内生归责：油价是外生冲击（来自 OPEC / 地缘）——选民对总统党归责弱。关税是内生政策——选民对总统党归责直接。1973/1979/1990/2022 油价 shock 都不让现任总统党 vote loss 等于 CPI 涨幅；但 2018-19 Trump-1 关税推 CPI 0.4pp 时选民对共和党归责显著（见 Autor-Dorn-Hanson-Majlesi 2020 关于关税地区与选票关联）。
  • 传导路径不同：油价主要冲 headline CPI（汽油 + 能源价格直接 passthrough）；关税主要冲 goods CPI（商品价格 passthrough）。CPI 篮子里 goods ~21%（vs 服务 ~58% + 食品 + 能源 ~21%），所以关税 passthrough 限定在 goods 部分。
  • 持续性不同：油价 shock 通常是 pulse（事件 + 衰减）；关税是 sustained level shift（政策不变则常态化）。这决定 CPI YoY 数字 vs CPI level 数字的政治含义完全不同。

具体校准

配置常量（pipeline/config.py）：
  TARIFF_AVG_RATE         = 0.135  # May 2026 effective avg tariff rate
  TARIFF_PRE_2025_RATE    = 0.025  # pre-Trump-2 baseline MFN
  TARIFF_TO_CPI_PP_PER_PP = 0.10   # passthrough elasticity (Cavallo 2021 conservative)

当前对 CPI 的贡献：
  tariff_cpi_pp_in_current_yoy = (13.5 - 2.5) × 0.10 = 1.1pp
  即当前 CPI YoY 3.3% 里有 ~1.1pp 是关税推升；
  "反事实无关税" CPI 应为 ~2.2%。

参考文献：
  - Cavallo et al (2021) AER：2018-19 Trump-1 关税近完整 passthrough (~0.95)
  - Amiti-Redding-Weinstein (2020)：类似结论
  - 当前 0.10 弹性是 conservative 估计——考虑 Trump-2 broader-based + margin absorption + FX 调整

已知局限

  • 0.10 弹性偏保守。Cavallo (2021) 在 Trump-1 估计接近 ~0.25（近完整 passthrough），但 Trump-2 broader-based 让 margin absorption + FX 调整效应增大。真实弹性可能在 0.10-0.25 区间。读 dashboard 时 tariff sensitivity 数字应该理解为下限。
  • 没有对总统党归责的额外加成。Lewis-Beck 模型里关税推升的 CPI 跟油价推升的 CPI 等同对待。理论上 voter attribution 应该让关税通道有额外 -0.5 到 -1pp 的归责惩罚——这部分未建模。
  • 没有 goods vs 服务 CPI 分解。当前用 headline CPI YoY 单一变量。如果 dashboard 加 PCE / Core PCE / Goods CPI 分项追踪，关税信号可以更精确分离。

七、B3 backtest 说明（2026-05 PR 新增）

pipeline/processors/backtest.py 是新加的 B3 backtest 模块——为 ensemble RMSE 计算提供独立的协方差矩阵 + 给四个模型的 paper-reported coefficients 做出样本验证。具体见章节二已展示的结果。本节说明数据集 + 方法学。

数据集（HISTORICAL_ELECTIONS, n=11）

当前 PR 包含 1980-2020 这 11 次选举的 hand-assembled 数据。每行有 12 个字段：incumbent_party / open_seat / inc_two_party_pct / june_approval_net / dper / dur / term2 / G / P / Z / R / G_Q2 / UNRATE_avg / CPI_YoY，加上 confidence 标记和 provenance 注释。

数据源：

  • incumbent_two_party_pct：David Leip's Atlas / 官方认证结果（公开记录）
  • june_approval_net：Gallup 历史档案（选举年 6 月最接近的民调）
  • G / P / Z（Fair 输入）：Ray Fair 自己发布的 fairmodel.econ.yale.edu spreadsheet
  • R（Hibbs 输入）：Douglas Hibbs 2008 BBV2008.xlsx
  • G_Q2（Abramowitz 输入）：BEA NIPA Table 1.1.1 quarterly real GDP growth
  • UNRATE_avg / CPI_YoY：BLS U-3 年均 + CPI-U 10 月值

为什么不包含 1948-1976

这 7 次选举（1948, 52, 56, 60, 64, 68, 72, 76）暂时排除，confidence='low'。原因：

  • archival GDP per capita 季度细粒度从训练数据回忆 confidence 较低
  • Gallup 6 月民调对 1950s-60s 时段不连续
  • Hibbs 自己的 BBV2008 spreadsheet 包含这些 quarterly RDPI 但我没 verified 到具体表号

扩展到 18 选举完整样本是单 PR 的事——只需要填补 HISTORICAL_ELECTIONS list 里的 7 行，重跑 backtest 即可。预期 n=18 + 4 models × 1-6 params 让自由度从 5-10 涨到 12-17，OLS 估计稳定性显著上升。

方法学：OLS via numpy.lstsq

每个模型独立 OLS（不联合估计——四模型是独立预测器，不是 SUR 系统）：

  Fair:        V_inc ~ G + (-P) + Z + dper·sign + (-dur)  [6 params + intercept]
  Hibbs:       V_inc ~ R                                  [1 param + intercept]
  Abramowitz:  V_inc ~ G_Q2 + june_approval + term2        [3 params + intercept]
  Lewis-Beck:  V_inc ~ -(UNRATE + CPI_YoY)                 [1 param + intercept]

残差协方差矩阵：
  R = [r_fair, r_hibbs, r_abramowitz, r_lewis_beck]ᵀ  (4 × n 矩阵)
  Σ = R Rᵀ / (n - 1)
  corr_ij = Σ_ij / sqrt(Σ_ii · Σ_jj)

修正后的 ensemble RMSE：
  Var(集合) = wᵀ Σ w  （独立性假设忽略了 off-diagonal）
  RMSE_correlated = sqrt(Var(集合))

为什么 backtest 不自动改 config.py

  • n=11 太小（自由度 5-10），Fair β_dper 在 OLS 输出符号翻转，是 small-sample artefact
  • 2020 COVID Q2 -31.4% 是 extreme outlier，Abramowitz 系数 OLS 估计严重失真
  • Lewis-Beck 是 2026-05 PR 新加，paper-reported priors 只是 2 点（1980/2010）反推——OLS 估计虽然有 sample 基础，但 n=11 + outlier 影响下不应替换

结论：backtest 当前的角色是诊断工具，不是 model update pipeline。读者看 paper RMSE 与样本 RMSE 的 gap，做出"95% CI 应放宽 ×2.8 倍"的修正解读；具体系数仍使用 paper-reported 数值。完整 1948-2020 18 选举样本上线后，可以考虑用 OLS 估计替换 priors。

十、Dashboard 的能力边界

  • 不输出 2028 大选州层预测。候选人未定（特朗普任期满，开放席位）、初选未开始、2.5 年外州层数据噪音远大于信号。任何具体 2028 州层数字均属假精度。
  • 不输出 2026 众议院选区层预测。选区级预测需 Cook PVI 选区数据 + 候选人质量 + 现任优势变量，需订阅 Cook Political Report。本 dashboard 仅提供州层估算。
  • 不输出"X% 概率赢 2028"单点预测。三模型集合 RMSE ±2.5pp，加上 2.5 年外的额外不确定性，单点估计具误导性。
  • 不预测非经济议题影响。如 Dobbs 判决（堕胎议题）等非经济变量不在任何模型内。该类变量在 2022 中期帮助民主党抵消油价冲击；2026 类似变量可能再现，但模型无法捕捉。
  • 不预测黑天鹅事件。新一轮 911 级恐袭、大型自然灾害、重大金融崩盘等均不在传导链内。

1. Why these four models? Why not others?

The US election forecasting field has approximately 6-8 academically-backed models. This dashboard selects four complementary models (Fair / Hibbs / Abramowitz / Lewis-Beck-Tien) and constructs an ensemble forecast. Lewis-Beck was added in the 2026-05 PR as the 4th model — the first three are pocketbook/retrospective tradition (GDP growth, real income, approval), all endogenous to macroeconomic variables; Lewis-Beck adds the sociotropic dimension ("national unemployment + inflation"), providing an independent signal for unemployment-dominated midterm scenarios like 1982/2010. Reasons:

Explicit formulas of the four selected models

1. Fair Vote Equation (Ray Fair, Yale 1978), predicts the in-party two-party vote share:

V_Democrat = 47.42 + 0.667·G·I + (-0.690)·P·I + 0.984·Z·I
        + 3.41·DPER + (-3.46)·DUR·I + (-0.105)·I

Variables:
  G    = real per-capita GDP growth, election-year Q1-Q3, annualized %
  P    = |GDP deflator growth| over administration's first 15 quarters, annualized %
  Z    = number of "good news quarters" (real per-capita growth >3.2%) in first 15 quarters (max 15)
  DPER = sitting president personally running (D=+1, R=-1, open seat=0)
  DUR  = duration penalty (first term=0, second consecutive=1, third=1.25...)
  I    = party dummy (D=+1, R=-1)
RMSE = ±2.5pp (national two-party vote, out-of-sample)

2. Hibbs Bread & Peace (Douglas Hibbs, 1982/2008), predicts the in-party two-party vote share:

V_incumbent = 46.10 + 3.49·R + (-0.00355)·cumulative war fatalities

Variables:
  R = λ-weighted real per-capita disposable income (RDPI) growth over 15 quarters, annualized %
  λ = 0.915 (geometric quarterly decay — most recent quarter weighted highest)
  Cumulative war fatalities = US military deaths in non-defensive wars during the term (0 for 2026-2028)
RMSE = ±1.85pp (national two-party vote, 1952-2012 historical sample)

3. Abramowitz Time-for-Change (Alan Abramowitz, 1988), predicts the in-party two-party vote share:

V_incumbent = 51.40 + 0.108·G_Q2 + 0.642·NETAPP + (-4.0)·TERM2

Variables:
  G_Q2    = real GDP growth in election-year Q2, annualized %
  NETAPP  = sitting president's June net approval (approve - disapprove)
  TERM2   = dummy for in-party serving 2+ consecutive terms (yes=1, no=0)
RMSE = ±1.90pp

4. Lewis-Beck-Tien Sociotropic Misery Index (added 2026-05), predicts the in-party two-party vote share:

V_incumbent = 53.0 - 0.85 · (UNRATE + CPI_YoY)

Variables:
  UNRATE  = U-3 unemployment rate (%)
  CPI_YoY = headline CPI year-over-year inflation (%)
  misery  = UNRATE + CPI_YoY ("misery index" sum, classic Lewis-Beck-Tien 1996/2008 form)
RMSE = ±2.50pp ([CALIBRATION_PENDING] — current coefficients are 2-point priors from 1980/2010, not OLS-fit on the full 1948-2020 sample)

Why included: Fair/Hibbs/Abramowitz are all pocketbook tradition (individual income), unable to capture 1982 (UNRATE 10.8% / CPI 3.8%) and 2010 (UNRATE 9.6% / CPI 1.1%) midterms where unemployment dominated. Lewis-Beck is sociotropic tradition (national economic news), adding an independent unemployment dimension. See Mutz (1992) and Lewis-Beck & Tien (1996, 2008) for the pocketbook vs sociotropic voting literature.

5. Bafumi-Erikson-Wlezien (BEW, 2010), predicts midterm House seat change:

In-party seat change = -22 + 3.5·in-party Generic Ballot lead + 0.4·sitting president's net approval

Variables:
  In-party Generic Ballot lead = Generic Ballot for in-party - opposition, in pp
  Sitting president's net approval = approval - disapproval, in pp
  -22 intercept = historical average midterm in-party seat loss
RMSE = ±12 seats

Selection principles summarized: (1) Academic backing + publicly available formulas; (2) Diverse input variables (comprehensive, income, approval) to maximize ensemble complementarity; (3) Currently available data (no dependence on yet-to-be-released primary data); (4) Out-of-sample RMSE ≤ 2.5pp.

2. Ensemble methodology, known limitations, and B3 backtest correction

Principle: Single models always have biases; combining complementary models reduces average error. The PollyVote method, proposed by German researcher Andreas Graefe, has shown that weighting all public models by inverse RMSE (smaller historical error → larger weight) produces an out-of-sample RMSE 30-40% lower than any single model.

Specific calculation used in this dashboard (2026-05 PR upgraded to 4-way ensemble):

Weight w_i = (1 / RMSE_i) / Σ(1 / RMSE_j)
Ensemble forecast = Σ w_i × Model_i prediction
Ensemble variance = Σ w_i² × RMSE_i² (assuming independence — see ⚠️ below)

Current weights (4-way ensemble, inverse paper-reported RMSE):

  • Hibbs ≈ 29% (paper RMSE 1.85)
  • Abramowitz ≈ 28% (paper RMSE 1.90)
  • Fair ≈ 21% (paper RMSE 2.50)
  • Lewis-Beck ≈ 21% (paper RMSE 2.50 [CALIBRATION_PENDING])

Under independence assumption, ensemble band ≈ ±1.07pp (per production code).

⚠️ Known issue with this calculation: the variance formula assumes four-model independence, but Fair / Hibbs / Abramowitz all share overlapping macroeconomic variables (GDP growth, inflation, income); Lewis-Beck and Hibbs hit common outliers in unemployment-cycle years. Model prediction errors are positively correlated, not independent.

Why backtest does NOT auto-update config.py coefficients

  • n=11 is too small — with 4 models × 1-6 parameters each, df is 5-10. Estimates are noisy. Fair β_dper flips sign in OLS output (3.41 → -6.82) — this is small-sample + collinearity artefact, not a real signal.
  • 2020 COVID is an extreme outlier — Q2 GDP -31.4% severely distorts Abramowitz coefficient estimation. A full 1948-2020 18-election sample would be more balanced.
  • Current implementation: backtest writes to data/election/backtest_results.json (manual trigger only); the methodology panel displays comparison data so readers see the paper-RMSE vs sample-RMSE gap, but production ensemble still uses paper-reported coefficients.
  • Extension path: fill in the 7 pre-1980 elections in HISTORICAL_ELECTIONS (requires Hibbs 1982 spreadsheet + Fair archive + Gallup historical), then a single PR can graduate to OLS-fit coefficients.

Model disagreement is itself a signal

When all four models agree (strong input signal), ensemble confidence is high. When they diverge sharply (currently Fair vs Abramowitz differ by 11pp), it indicates "models give different answers because they weight inputs differently" — this itself is a diagnostic signal, indicating economic data and voter perception are sharply divergent. 1992 / 2016 / 2024 elections were all such cases. The Fair vs Abramowitz gap shown on the 2026 dashboard is an attention signal, not pure noise.

Why ICS↔approval is flagged parallel, not one-way chain

The original 9-step chain treated step 7 (ICS) → step 8 (approval) as one-way causation (-10 ICS → -1.1pp approval). This is misleading:

  • Erikson, MacKuen & Stimson (2002) The Macro Polity Chapter 4 uses VAR analysis to show that approval → ICS Granger-causes more strongly than the reverse (political sentiment spills over to economic perception); both are mainly driven by basement-level fundamentals (real income + unemployment + price level).
  • De Boef & Kellstedt (2004) similarly argue that modeling ICS → approval as one-way causation double-counts the same shock.
  • 2026-05 PR fix: transmission_state.json adds a parallel_group field; the frontend chain tab marks steps 7+8 with a purple top border + "‖" badge to flag parallel, plus an indigo notice explaining why they should be read as co-driven signals rather than chain steps.
  • Explaining Hibbs's 1992 / 2016 / 2024 failures: in these three elections ICS and RDPI diverged sharply — RDPI fine but ICS dropped (1992 Bush 41 / 2024 Biden) or RDPI weak but ICS rose (2016 Clinton). When perception and income diverge, Hibbs (using RDPI) systematically misses — making the perception-basement dual-signal observation necessary.

4. Additional transmission channels under military intervention (scenario D)

Section 3's nine-step transmission chain covers economic channels — oil prices propagate through inflation and real income to voter perception. But scenario D (US Navy directly enforces the strait) involves military action, activating two completely independent non-economic channels. This section explains how these channels work and why "military victory" affects the Senate, House, and presidential election quite differently.

Channel A: Rally-around-the-flag (patriotic surge)

Mechanism: When the US conducts major military action, public sentiment briefly converges in patriotic support, producing an instantaneous positive shock to presidential approval — even among voters who don't normally support the president.

Historical calibration (from Mueller's 1973 classic War, Presidents and Public Opinion):

  • 1979 Iran hostage crisis: Carter approval +18pp
  • 1991 Desert Storm: Bush 41 approval rose from 60% to 89% (+29pp peak)
  • 2001 9/11: Bush 43 approval +35pp (historical maximum)

For "successful escort/strait control" we use +8pp (median estimate for small-scale military operations, far below full-scale war).

Key feature: rally is a pulse signal that decays. Mueller measured a half-life of approximately 6 months — meaning each 6-month interval cuts remaining strength in half:

Months past eventRemaining strengthApproval bump
0 (event just occurred)100%+8pp
6 (one half-life)50%+4pp
12 (two half-lives)25%+2pp
30 (2028 election)3%+0.2pp (essentially gone)

Historical mirror: Bush 41's approval after Desert Storm was 89%, but fell to 34% within 18 months — Clinton thus won the 1992 election. This was the Fair model's most famous failure: economic data looked acceptable, the model predicted Bush winning at 55%, but the rally had long faded and voters only remembered the recession.

Channel B: Casualty drag

Mechanism: Long-term military deployment inevitably produces US military casualties. Each fatality accumulates as a negative weight in voter memory. Hibbs Bread & Peace formally captures this: each fatality ≈ -0.00355pp vote share (coefficient from 1952-2012 historical regression).

Historical calibration:

  • Bush 41 Gulf War: ~150 US fatalities (short duration, limited effect)
  • Bush 43 Afghanistan war: cumulative ~2400 US fatalities (dragged on R in 2008/2012)
  • Hormuz escort operation: estimated far below war scale; this dashboard uses -3pp long-term drag for scenario D

Key feature: casualty is a cumulative signal that does NOT decay. Each fatality enters the permanent ledger and is never forgotten:

Months past eventCumulative ratioApproval drag
00%0pp
6 (2026 midterm)50%-1.5pp
30 (2028 presidential)100%-3pp (full)

Combining both channels = the time scissors

Stacking decaying rally and accumulating casualty drag yields a counterintuitive curve: military intervention helps in the short-term election but actually hurts in the long-term election. For the dashboard's two horizons:

HorizonRally remainingCasualty accumulatedNet approval impact
2026 midterm (6 months out)+4.0pp (50% remaining)-1.5pp (half accumulated)+2.5pp (R benefits)
2028 presidential (30 months out)+0.2pp (essentially zero)-3.0pp (fully accumulated)-2.8pp (R hurt)

Comparison to baseline (B drag): under D control, oil is $90/barrel (vs B baseline's $100), so the economic channel helps Republicans (lower oil → lower inflation → better real income). But the rally-decay + casualty-accumulation non-economic channels offset this economic benefit.

Why the effect differs between Senate, House, and presidential

The same approval shift produces different responses at the three levels because input variables, response formulas, and structural constraints all differ:

Counterintuitive finding: D control's percentage effect on the Senate (D majority probability -7pp) is more visible than on the House. This is because the Senate has only 33 races, and the swing seats (ME / NC / TX) are at the margin — a small approval bump can flip 1-2 seats relative to a 33-seat denominator. The House has 435 seats, so a few-pp approval shift gets diluted in the structural noise.

Model limitations

  • Both rally and casualty channels affect all 35 Senate/House race candidates uniformly — the model assumes candidates are foreign-policy neutral. In reality, candidates have varied stances (hawk vs dove) and could produce differentiated effects.
  • The +8pp rally coefficient is a median estimate; the sensitivity sweep covers +5 to +15 but does not include +24pp 1991 GW peak — a clean fast win with minimal casualties could produce +24pp.
  • The -3pp casualty coefficient assumes "successful escort." The high_casualty calibration uses -8pp, but Iraq-scale deployment could push to -10pp or worse.
  • This dashboard does not model candidate quality, primary mobilization, advertising war, or debate performance — these may amplify or offset the transmission effects above.
  • casualty_sensitivity is computed only for Hormuz scenarios that have rally_effect_pp configured (currently just D_us_control); E_total_war should also activate it but isn't yet configured.

5. Time dimension: why the Iran war affects 2026 midterms little but 2028 presidential a lot

This is an important dashboard finding that's easy to miss: the 5 Hormuz scenarios produce small variation in 2026 midterm outcomes but huge variation in 2028 presidential outcomes. The asymmetry table:

Election5-scenario rangeInterpretation
2026 House R seat loss-36 to -40 (4-seat range)Almost identical
2026 Senate D net seat gain+3.7 to +4.0 (0.3-seat range)Almost identical
2026 Senate D majority probability53% to 60% (7pp range)Small variation
2028 Presidential R two-party share44.7% to 49.7% (5pp range)Much wider variation

Three reasons

Reason 1: 2026 midterm fundamentals are already locked in. Regardless of how the Iran war evolves, these indicators are already showing on the dashboard:

  • Trump net approval -13.5 (in alarm zone)
  • Michigan Consumer Sentiment 53.3 (near 1980 / 2008 / 2022 crisis bottoms)
  • Generic Ballot D+3.5 (already showing R disadvantage at midterm)

These indicators have already priced in "Trump's first-year performance + the partial transmission of the current Hormuz shock." Even if the Iran conflict resolved tomorrow (scenario A), Republicans would still lose ~40 House seats in 2026. The political damage has been done.

Reason 2: Transmission lag is 4-9 months — most won't reach the November midterm. From May 8 to November 3, 2026 is only ~6 months, but full oil-shock transmission requires:

  • Oil prices (step 1) → already in place
  • CPI (step 4) → won't fully reflect until July-August
  • Real income (step 6) → September data at earliest
  • Consumer sentiment (step 7) → already collapsed, but as a result of past shocks, not new ones

Most damage that the midterm can "absorb" is already absorbed. Any new oil shock between May and November has limited marginal impact on the November vote.

Reason 3: Senate map's hard structural ceiling. Class II Senate map has 22 R defending vs 11 D defending. Democrats can net at most 2-4 seats. This is a structural physical constraint, unaffected by any oil scenario.

The real "Iran war effect" lives in 2028

Look at the 2028 R two-party numbers carefully: A_easing 49.7% → E_total_war 44.7% = 5pp range. This corresponds to:

  • 49.7%: Republicans nearly tied with Democrats (hypothetical EC margin <50 votes)
  • 44.7%: Republicans suffer a 1980 Carter / 2008 McCain-level defeat (EC margin 200+ votes)

Two completely different political worlds. This is what the Iran war is actually wagering.

What this means for analysis

  • Stop asking "how much will the Iran war affect the 2026 midterms" — the answer is "very little, because the fundamentals are already set."
  • Ask instead: How did current -13.5 approval + 53 ICS form? What indicators will update in June, September, October? What is the 5pp range for 2028 actually wagering?
  • In short: 2026 midterm has already become a baseline of "Democrats likely take House + Senate stalemate." The Iran war is now a 2028 story, no longer a 2026 story.

A blind spot to watch

The model cannot capture non-economic mobilization effects. The 1979 Iran hostage crisis briefly helped Carter via patriotic rally (+18pp). If 2026 sees large-scale US military casualties, US homeland attacks, or a 9/11-level event, the entire picture would be rewritten — these aren't in the transmission chain. If a qualitative change (not continuation but a new event) occurs in the next 6 months, all numbers must be recalculated.

6. Tariff channel (independent of oil shock, added 2026-05 PR)

The original dashboard treated all inflation shocks as exogenous (coming from the Hormuz oil situation). But the real dominant inflation shock in 2025-2026 is not oil — it's the Trump second-term tariffs, a channel the dashboard previously missed entirely. The 2026-05 PR adds tariff as an independent channel in the macro projection.

Why tariff requires independent modeling (cannot be merged into the oil channel)

  • Exogenous vs endogenous attribution: oil is an exogenous shock (OPEC / geopolitics) — voters attribute weakly to the incumbent party. Tariffs are an endogenous policy choice — voters attribute directly to the incumbent. The 1973/1979/1990/2022 oil shocks did not produce vote losses equal to the CPI bump; but 2018-19 Trump-1 tariffs that raised CPI 0.4pp produced significant Republican vote loss (see Autor-Dorn-Hanson-Majlesi 2020 on tariff regions vs vote behavior).
  • Different transmission paths: oil mainly hits headline CPI (gasoline + energy direct passthrough); tariffs mainly hit goods CPI (commodity-price passthrough). Goods is ~21% of the CPI basket (vs services ~58% + food/energy ~21%), so tariff passthrough is limited to the goods component.
  • Different persistence: oil shocks are typically pulses (event + decay); tariffs are sustained level shifts (policy in force = permanent). This makes the political meaning of CPI YoY vs CPI level fundamentally different.

Specific calibration

Config constants (pipeline/config.py):
  TARIFF_AVG_RATE         = 0.135  # May 2026 effective average tariff rate
  TARIFF_PRE_2025_RATE    = 0.025  # pre-Trump-2 baseline MFN
  TARIFF_TO_CPI_PP_PER_PP = 0.10   # passthrough elasticity (Cavallo 2021 conservative)

Current CPI attribution:
  tariff_cpi_pp_in_current_yoy = (13.5 - 2.5) × 0.10 = 1.1pp
  i.e., of current CPI YoY 3.3%, ~1.1pp is tariff-driven;
  counterfactual "no-tariff" CPI would be ~2.2%.

References:
  - Cavallo et al (2021) AER: 2018-19 Trump-1 tariffs near-complete passthrough (~0.95)
  - Amiti-Redding-Weinstein (2020): similar conclusion
  - Current 0.10 elasticity is conservative — accounts for Trump-2 broader-based + margin absorption + FX adjustment

The 2026-05 PR runs each Hormuz scenario under 3 additional tariff variants, exposing sensitivity to readers:

Tariff elasticity effect is an order of magnitude smaller than oil scenarios — because Hibbs uses RDPI not directly tied to CPI, only Fair / Abramowitz / Lewis-Beck in the ensemble are affected through CPI (Lewis-Beck directly, Fair through P, Abramowitz through approval via ICS via CPI).

Known limitations

  • 0.10 elasticity is conservative. Cavallo (2021) found ~0.25 for Trump-1 (near-complete passthrough), but Trump-2's broader-based nature increases margin absorption + FX adjustment effects. True elasticity likely in 0.10-0.25 range. The tariff sensitivity numbers on the dashboard should be read as a lower bound.
  • No additional attribution bonus to the incumbent party. In the Lewis-Beck model, tariff-driven CPI is treated identically to oil-driven CPI. Theoretically voter attribution should give the tariff channel an additional -0.5 to -1pp attribution penalty — this part is unmodeled.
  • No goods vs services CPI decomposition. Currently uses headline CPI YoY as a single variable. If the dashboard added PCE / Core PCE / Goods CPI tracking, tariff signal could be more precisely isolated.

7. B3 backtest documentation (added 2026-05 PR)

pipeline/processors/backtest.py is the newly added B3 backtest module — providing an independent covariance matrix for ensemble RMSE computation + out-of-sample validation for the four models' paper-reported coefficients. Results are shown in section 2. This section documents the dataset + methodology.

Dataset (HISTORICAL_ELECTIONS, n=11)

The current PR includes hand-assembled data for 11 elections from 1980-2020. Each row has 12 fields: incumbent_party / open_seat / inc_two_party_pct / june_approval_net / dper / dur / term2 / G / P / Z / R / G_Q2 / UNRATE_avg / CPI_YoY, plus confidence flag and provenance notes.

Data sources:

  • incumbent_two_party_pct: David Leip's Atlas / official certified results (public record)
  • june_approval_net: Gallup historical archive (June poll closest to election year)
  • G / P / Z (Fair inputs): Ray Fair's own published spreadsheet at fairmodel.econ.yale.edu
  • R (Hibbs input): Douglas Hibbs 2008 BBV2008.xlsx
  • G_Q2 (Abramowitz input): BEA NIPA Table 1.1.1 quarterly real GDP growth
  • UNRATE_avg / CPI_YoY: BLS U-3 annual average + CPI-U October value

Why 1948-1976 is not included

These 7 elections (1948, 52, 56, 60, 64, 68, 72, 76) are temporarily excluded, confidence='low'. Reasons:

  • archival quarterly GDP per capita granularity is harder to recall from training data with high confidence
  • Gallup June poll for 1950s-60s period is non-continuous
  • Hibbs's own BBV2008 spreadsheet contains these quarterly RDPI values but I haven't verified to specific tables

Extending to the full 18-election sample is a single-PR diff — just filling 7 rows in HISTORICAL_ELECTIONS and re-running backtest. n=18 + 4 models × 1-6 params raises df from 5-10 to 12-17, substantially improving OLS stability.

Methodology: OLS via numpy.lstsq

Each model fit independently (no joint estimation — the four models are
independent predictors, not an SUR system):

  Fair:        V_inc ~ G + (-P) + Z + dper·sign + (-dur)  [6 params + intercept]
  Hibbs:       V_inc ~ R                                  [1 param + intercept]
  Abramowitz:  V_inc ~ G_Q2 + june_approval + term2        [3 params + intercept]
  Lewis-Beck:  V_inc ~ -(UNRATE + CPI_YoY)                 [1 param + intercept]

Residual covariance matrix:
  R = [r_fair, r_hibbs, r_abramowitz, r_lewis_beck]ᵀ  (4 × n matrix)
  Σ = R Rᵀ / (n - 1)
  corr_ij = Σ_ij / sqrt(Σ_ii · Σ_jj)

Corrected ensemble RMSE:
  Var(ensemble) = wᵀ Σ w  (independence assumption ignores off-diagonal)
  RMSE_correlated = sqrt(Var(ensemble))

Why backtest does NOT auto-update config.py

  • n=11 too small (df 5-10), Fair β_dper flips sign in OLS output — small-sample artefact
  • 2020 COVID Q2 -31.4% is an extreme outlier, severely distorting Abramowitz coefficient estimates
  • Lewis-Beck was newly added in 2026-05 PR; paper-reported priors are just 2 points (1980/2010) — OLS estimates have sample basis but under n=11 + outliers shouldn't replace them

Conclusion: backtest's current role is as a diagnostic tool, not a model-update pipeline. Readers see paper-RMSE vs sample-RMSE gap, derive the "widen 95% CI by ×2.8" correction; specific coefficients still use paper values. When the full 1948-2020 18-election sample lands, OLS estimates could replace priors.

8. Calibration sources for transmission elasticities

Four historical oil-shock events serve as calibration anchors:

  • 1980 Iranian Revolution + Iran-Iraq War: oil $14→$39 (+179%), CPI peak 14.8%, ICS 75→51, Carter two-party share 44.7% — catastrophic loss. Most direct historical analogue to the current Hormuz crisis — same Persian Gulf origin, same long duration, and ultimately complete voter sentiment collapse.
  • 1990-91 Gulf War: oil $17→$41 (+141%), but only lasted 5 months. Bush 41 ultimately lost to Clinton — proving that brief oil shocks can have political aftereffects lasting 18+ months. Also Fair model's most famous failure case.
  • 2008 oil peak: oil $60→$147 (+145%), lasted 8 months. McCain lost by 7.4pp. Proves the destructive power of simultaneous economic data and voter perception deterioration.
  • 2022 Russia-Ukraine war: oil $90→$128, offset by Dobbs ruling. Proves cumulative inflation memory can persist into the next election cycle (Harris 2024 still bore the burden).

10. Dashboard capability boundaries

  • Does not output 2028 presidential state-level forecasts. Candidates undecided (Trump term-limited, open seat); primaries haven't begun; state-level data noise vastly exceeds signal 2.5 years out. Any specific 2028 state-level number is false precision.
  • Does not output 2026 House district-level forecasts. District-level forecast requires Cook PVI by district + candidate quality + incumbency variables, which require Cook Political Report subscription. The dashboard provides only state-level estimates.
  • Does not output single-point predictions like "X% probability R wins 2028". Three-model ensemble RMSE is ±2.5pp, plus additional uncertainty from being 2.5 years out — single-point estimates are misleading.
  • Does not predict non-economic issue effects. E.g., the Dobbs ruling (abortion) — non-economic variables are not in any model. That variable helped Democrats offset the oil shock in 2022 midterms; similar variables may reappear in 2026, but the models cannot capture them.
  • Does not predict black swan events. New 9/11-scale terror attacks, major natural disasters, major financial collapses — none are in the transmission chain.

11. Glossary

RMSE (Root Mean Squared Error)
The average magnitude by which a model's historical predictions deviate from true values. Fair model's RMSE of 2.5pp means: if it predicts the Republicans at 50%, the true value will be in the 45-55% range 95% of the time.
PVI (Cook Partisan Voting Index)
How a state or district leans relative to the national average. E.g., Maine's D+2 means Maine leans 2pp more Democratic than the national average.
ICS (Michigan Index of Consumer Sentiment)
The University of Michigan's monthly consumer sentiment survey, indexed to 1966=100. The current 53 is near 1980/2008 crisis-bottom levels.
Generic Ballot
"Regardless of candidate, will you vote Democrat or Republican in the next congressional election" — the polling-aggregated value. Core input to the Bafumi-Erikson-Wlezien model.
DPER
Fair model variable: sitting president personally running = +1 (D) / -1 (R), open seat = 0. R in 2028 = 0 (Trump term-limited), losing approximately 3-4pp incumbency advantage.
DUR
Fair model "duration penalty" variable: first term = 0, two or more consecutive terms ≥ 1. R in 2028 has only one term, so DUR = 0.
Z (Good News Quarters)
Fair model variable: number of quarters in the first 15 quarters of the administration where real per-capita GDP growth exceeded 3.2%.
Class II / Class III Senate
US senators serve 6-year terms divided into three rotating classes. 2026 elects Class II (22 R + 11 D); 2028 elects Class III (~21 R + 12 D).
Bafumi-Erikson-Wlezien (BEW)
The 2010 midterm House seat forecasting model by these three scholars, with Generic Ballot as core input variable.
